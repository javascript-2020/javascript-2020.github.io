x509 Certificates in Javascript : Encrypt / Decrypt Data

26 Apr 2026

This blog post describes using x509 certificates to encrypt and decrypt data in javascript environments.
This can be useful since the public certificate can be made available publicly, even sent via an insecure communication channel, this public certificate can be used to encrypt data, the encrypted data can be sent to the holder of the private key, who can subsequently decode the encrypted data using the private key.
If two parties want to have a bi-directional encrypted communication, they can both send each other their public certificates.

SPKI

SPKI in X.509 stands for Subject Public Key Info.
It is the part of an X.509 certificate that contains the subject’s public key and information about the algorithm used with that key.

What SPKI actually contains

The Subject Public Key Info (SPKI) field is a structured ASN.1 block inside every X.509 certificate. It includes:

  • The subject’s public key (e.g., RSA, ECDSA, Ed25519)
  • The algorithm identifier for that key (e.g., rsaEncryption, id-ecPublicKey)
  • Any algorithm parameters required by that key type

This is the part of the certificate that clients use to verify signatures or establish encrypted sessions.

Why SPKI matters

SPKI is essential because:

  • It is the actual public key that the certificate binds to an identity.
  • It is used by TLS, code signing, S/MIME, and other PKI systems to verify authenticity.
  • It is the part that gets hashed when generating a certificate pin (HPKP-style pins or modern SPKI pinning in some systems).

In short, SPKI is the cryptographic heart of the certificate.

Where SPKI sits in the certificate structure

According to the X.509 standard, SPKI is one of the core Version 1 certificate fields, meaning it has existed since the earliest form of the standard.

Typical X.509 structure (simplified):

  • Version
  • Serial Number
  • Signature Algorithm
  • Issuer
  • Validity
  • Subject
  • Subject Public Key Info (SPKI)
  • Extensions (v3)
  • Signature
